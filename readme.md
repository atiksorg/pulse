# 📊 Pulse (Events Analytics)

**Pulse** — это минималистичный и сверхбыстрый сервис для сбора событий и построения дашбордов. Создан для разработчиков, которым нужна аналитика прямо сейчас, без сложной настройки баз данных, регистрации и тяжеловесных SDK.

Один HTTP-запрос для отправки события, два клика для создания красивого графика.

![Pulse Dashboard Preview](https://via.placeholder.com/1000x500.png?text=Место+для+вашего+скриншота+дашборда)

## ✨ Ключевые возможности

*   **Zero-Setup интеграция:** Отправляйте события простым `GET` или `POST` запросом. Не нужно создавать таблицы — база подстроится под ваш JSON payload.
*   **Сверхбыстрый бэкенд:** Написан на чистом Node.js (без фреймворков) поверх `better-sqlite3`. Использует WAL-журналирование, шардирование таблиц по месяцам и адаптивную batch-запись в памяти. Легко держит тысячи RPS на слабом VPS.
*   **No-Code Дашборды:** Конструктор метрик (Line, Bar, Pie, KPI, Таблицы, Логи). Поддержка Drag & Drop (Сетка) и режима свободного позиционирования (Холст).
*   **✨ AI-Помощник:** Интегрированная LLM. Просто напишите *"Покажи выручку по дням за неделю"* — и ИИ сам создаст нужный график с правильными агрегациями и фильтрами.
*   **Безопасность и Приватность:** Никаких email и паролей. Доступ в кабинет по уникальному идентификатору источника (`src`) и 4-значному PIN-коду. JWT сессии живут локально в оперативной памяти и `sessionStorage`.
*   **Публичные ссылки:** Делитесь дашбордами с командой или клиентами (Read-only режим). Доступ можно отозвать в любой момент.

## 🚀 Быстрый старт (Отправка событий)

Вам не нужно устанавливать библиотеки. Просто отправьте запрос на сервер:

**JavaScript (Браузер / Node.js):**
```javascript
// Отправка покупки
fetch("https://events.atiks.org/e?src=my_app&type=purchase&amount=1500&currency=rub");

// Отправка клика
fetch("https://events.atiks.org/e?src=my_app&type=click&button=hero_cta");
```

**Python:**
```python
import requests
requests.get("https://events.atiks.org/e", params={
    "src": "my_app",
    "type": "error",
    "code": 500,
    "message": "DB timeout"
})
```

**Отправка сырого JSON (POST):**
```bash
curl -X POST https://events.atiks.org/e \
     -H "Content-Type: application/json" \
     -d '{"src":"my_app", "type":"signup", "plan":"pro", "country":"kz"}'
```

После отправки перейдите на веб-интерфейс, введите ваш `src` (в данном случае `my_app`), придумайте 4-значный PIN-код и сразу стройте графики!

## 🛠 Установка (Self-Hosted)

Pulse не требует сборки фронтенда (No Webpack/Vite). Работает "из коробки".

### Требования
* Node.js v18+
* Python & Build tools (необходимы для компиляции `better-sqlite3` под вашу ОС).

### Запуск
```bash
# 1. Клонируйте репозиторий
git clone https://github.com/your-username/pulse.git
cd pulse

# 2. Установите зависимости
npm install better-sqlite3

# 3. Запустите сервер
node events_server.js
```
Сервер запустится на порту `3333`. UI будет доступен по адресу `http://localhost:3333/`.

### Переменные окружения (Environment Variables)

Вы можете настроить сервер перед запуском:

| Переменная | Описание | По умолчанию |
| :--- | :--- | :--- |
| `PORT` | Порт HTTP сервера | `3333` |
| `DB_PATH` | Путь к файлу SQLite | `./events.db` |
| `BATCH_SIZE` | Размер буфера событий перед записью в БД | `200` |
| `RETENTION_MONTHS` | Сколько месяцев хранить сырые данные | `6` |
| `ALLOWED_SRC` | Строгий список разрешенных `src` (через запятую) | *разрешены все* |
| `STATUS_TOKEN` | Токен (Bearer) для защиты эндпоинта `/status` | *без защиты* |

## 🏗 Архитектура проекта

Проект разделен на три логические части в минималистичном стиле:

*   **`events_server.js`** — Высоконагруженный шлюз приема событий. Реализует In-memory буферизацию (`adaptiveFlush`), партиционирование таблиц по датам (`events_YYYY_MM`) и агрегацию данных «на лету» (эндпоинт `/s`). Написан на модуле `http`.
*   **`auth.js` / `ai.js`** — Модули бизнес-логики: управление JWT сессиями, защита кабинетов (PIN Lockout), публичные Shares и интеграция с LLM-моделью.
*   **`/public`** — Vanilla JS фронтенд. 
    *   `core.js` — Сеть и In-memory State (никаких данных в `localStorage`).
    *   `dashboard.js` — Логика рендера графиков через `Chart.js`, таблиц, логов и Canvas-движок.
    *   `styles.css` — CSS-переменные, тёмная тема, адаптивная верстка под мобайл.

## 📡 API Справочник

### Запись данных
*   `GET /e` — Запись одного события (параметры в query).
*   `POST /e` — Запись одного события (параметры в JSON).
*   `POST /e/batch` — Пакетная запись (принимает массив JSON, до 1000 за раз).
*   `POST /e/clear` — Удаление всех событий пользователя (требует авторизации).

### Чтение данных (Агрегация)
*   `GET /s?src=..&type=..&group=..&agg=..` — Мощный агрегационный движок. 
    *   Поддерживает группировку по дате (`day`, `hour`), по полям payload (`group=field:country`).
    *   Поддерживает агрегации (`count`, `sum:amount`, `avg:duration`).
    *   Поддерживает фильтры, сортировку и лимиты.

### Инфраструктура
*   `GET /health` — Проверка состояния (200 OK, кол-во таблиц).
*   `GET /status` — Детальная сводка по оперативной памяти, размеру БД, скорости записи (может быть закрыт токеном).
*   `GET /export` — Выгрузка всех сырых событий в формате CSV (требует авторизации).
