# 📊 Pulse (Events Analytics)

Pulse — это минималистичный и сверхбыстрый сервис для сбора событий и построения дашбордов в реальном времени. Создан для разработчиков, которым нужна аналитика «здесь и сейчас», без сложной настройки баз данных, регистрации и тяжеловесных SDK. 

Один HTTP-запрос для отправки события, два клика для создания красивого графика, бесконечный холст для компоновки дашборда и встроенный скилл для ИИ-агентов.

---

## ✨ Ключевые возможности

- **Zero-Setup интеграция**: Отправляйте события простым `GET` или `POST` запросом. Не нужно создавать таблицы или схемы — база данных (SQLite) автоматически подстроится под ваш JSON payload.
- **Сверхбыстрый бэкенд**: Написан на чистом Node.js поверх `better-sqlite3`. Использует WAL-журналирование, шардирование таблиц по месяцам, адаптивную batch-запись в памяти и **Worker Threads** для тяжелых аналитических запросов (чтобы не блокировать прием событий).
- **Crash Recovery**: Встроенный WAL-буфер (`*.walbuf`) гарантирует, что события не потеряются даже при внезапном падении сервера до сброса буфера в БД.
- **No-Code Дашборды & Infinite Canvas**: Конструктор метрик (Line, Bar, Pie, KPI, Таблицы, Логи). Поддержка классической сетки (Drag & Drop) и режима **бесконечного холста** (Pan & Zoom) для свободного позиционирования панелей.
- **Готовые сценарии (Cases)**: Каталог шаблонов (E-commerce, SaaS, Error tracking, Product Analytics) с генерацией мок-данных в один клик.
- **Конструктор дашбордов (JSON)**: Возможность импортировать/экспортировать конфигурации дашбордов через JSON.
- **Скилл для ИИ-агентов**: Встроенный системный промпт, позволяющий LLM-агентам автономно отправлять события и генерировать ссылки на дашборды для пользователей.
- **Безопасность и Приватность**: Никаких email и паролей. Доступ в кабинет по уникальному идентификатору источника (`src`) и 4-значному PIN-коду. Защита от брутфорса (Lockout). Сессии живут локально в `sessionStorage`.
- **Публичные ссылки**: Делитесь дашбордами с командой или клиентами (Read-only режим). Доступ можно отозвать в любой момент.

## 🚀 Быстрый старт (Отправка событий)

Вам не нужно устанавливать библиотеки. Просто отправьте запрос на сервер:

**JavaScript (Браузер / Node.js):**
```javascript
// Отправка покупки
fetch("https://events.atiks.org/e?src=my_app&type=purchase&amount=1500&currency=rub");

// Отправка клика
fetch("https://events.atiks.org/e?src=my_app&type=click&button=hero_cta");
```

**Python:**
```python
import requests
requests.get("https://events.atiks.org/e", params={
    "src": "my_app",
    "type": "error",
    "code": 500,
    "message": "DB timeout"
})
```

**Отправка сырого JSON (POST) и пакетная запись:**
```bash
curl -X POST https://events.atiks.org/e \
     -H "Content-Type: application/json" \
     -d '{"src":"my_app", "type":"signup", "plan":"pro", "country":"kz"}'
```

После отправки перейдите на веб-интерфейс, введите ваш `src` (в данном случае `my_app`), придумайте 4-значный PIN-код и сразу стройте графики! 
*Совет: Используйте прямой вход по ссылке `https://events.atiks.org/src/my_app`, чтобы автоматически подставить ваш источник.*

## 🛠 Установка (Self-Hosted)

Pulse не требует сборки фронтенда (No Webpack/Vite). Работает "из коробки".

### Требования
- Node.js v18+
- Python & Build tools (необходимы для компиляции `better-sqlite3` под вашу ОС).

### Запуск
```bash
# 1. Клонируйте репозиторий
git clone https://github.com/atiksorg/pulse.git
cd pulse

# 2. Установите зависимости
npm install better-sqlite3

# 3. Запустите сервер
node events_server.js
```
Сервер запустится на порту `3333`. UI будет доступен по адресу `http://localhost:3333/`.

### Переменные окружения (Environment Variables)

| Переменная | Описание | По умолчанию |
|---|---|---|
| `PORT` | Порт HTTP сервера | `3333` |
| `DB_PATH` | Путь к файлу SQLite | `./events.db` |
| `BATCH_SIZE` | Размер буфера событий перед записью в БД | `200` |
| `BATCH_INTERVAL_MS` | Максимальный интервал между сбросами буфера | `1500` |
| `RETENTION_MONTHS` | Сколько месяцев хранить сырые данные (авто-удаление старых таблиц) | `6` |
| `ALLOWED_SRC` | Строгий список разрешенных `src` (через запятую) | *разрешены все* |
| `MAX_PAYLOAD_LENGTH` | Макс. длина JSON payload (в байтах) | `10000` |
| `STATUS_TOKEN` | Токен (Bearer) для защиты эндпоинта `/status` | *без защиты* |
| `TRUST_PROXY` | Доверять заголовку `X-Forwarded-For` (если за Nginx/Cloudflare) | `false` |

## 🏗 Архитектура проекта

Проект разделен на логические части в минималистичном стиле (Vanilla JS + Node.js):

### Бэкенд
- **`events_server.js`** — Высоконагруженный шлюз приема событий. Реализует In-memory буферизацию (`adaptiveFlush`), партиционирование таблиц по датам (`events_YYYY_MM`) и агрегацию данных «на лету». Написан на встроенном модуле `http`.
- **`analytics_worker.js`** — Пул Worker Threads для тяжелых SQL-запросов (агрегации, логи). Позволяет не блокировать основной Event Loop при чтении данных.
- **`auth.js`** — Модуль бизнес-логики: управление сессиями, защита кабинетов (PIN Lockout), дашборды и публичные Shares.
- **`ai.js`** — Интеграция с LLM. Парсинг ответов модели, валидация конфигов панелей, rate-limiting.

### Фронтенд (`/public`)
- **`core.js`** — Сеть, In-Memory State и утилиты (никаких данных в `localStorage`, только `sessionStorage`).
- **`dashboard.js`** — Логика рендера графиков (Chart.js), таблиц, логов, модальных окон и экспорта.
- **`interactive-canvas.js`** — Движок бесконечного холста (Pan/Zoom) для свободного размещения панелей.
- **`cases.js`** — Каталог готовых сценариев и генератор мок-данных.
- **`router.js`** / **`share.js`** / **`extras.js`** — Роутинг, рендер публичных ссылок, конфиг-билдер и визуальные эффекты.
- **`styles.css`** — CSS-переменные, тёмная тема, адаптивная верстка (Bottom Sheets, FAB, Scroll Shrink).

## 📡 API Справочник

### Запись данных (Events)
- `GET /e` — Запись одного события (параметры в query).
- `POST /e` — Запись одного события (JSON body).
- `POST /e/batch` — Пакетная запись (принимает массив JSON, до 1000 за раз).
- `POST /e/clear` — Удаление всех событий пользователя (требует авторизации).

### Чтение данных (Аналитика)
- `GET /s` — Мощный агрегационный движок.
  - **Группировка**: по дате (`day`, `hour`), по полям payload (`group=field:country`) или сырые логи (`group=raw`).
  - **Агрегации**: `count` (по умолч.), `sum:amount`, `avg:duration`.
  - **Фильтры**: `filters=[{"field":"country","op":"eq","value":"ru"}]` (поддерживает `eq`, `neq`, `gt`, `lt`, `in`, `contains`).
  - **Сортировка и лимиты**: `sort=value_desc`, `limit=10`.
- `GET /suggestions` — Возвращает уникальные типы событий и поля payload для автодополнения (требует авторизации).
- `GET /export` — Выгрузка всех сырых событий в формате CSV (требует авторизации).

### Дашборды и Шеринг
- `GET/POST /dashboards` — Список и создание дашбордов.
- `PUT/DELETE /dashboards/:id` — Обновление и удаление.
- `POST /dashboards/:id/share` — Создание публичной read-only ссылки.
- `GET /public/:share_id` — Получение конфига дашборда по публичной ссылке (без auth).
- `GET /shares` и `POST /shares/:id/revoke` — Управление активными ссылками.

### Авторизация и AI
- `POST /auth/register`, `/auth/login`, `/auth/logout` — Управление сессиями (PIN-код).
- `POST /ai/suggest-panel` — AI-помощник: принимает текстовый промпт («покажи выручку по дням») и возвращает валидный JSON-конфиг панели.
- `GET /src/:id` — Прямой вход в кабинет с предзаполненным `src`.

### Инфраструктура
- `GET /health` — Легкая проверка состояния (200 OK, размер буфера).
- `GET /status` — Детальная сводка по RAM, размеру БД, скорости записи, воркерам (может быть закрыт токеном).

## 🤖 Скилл для ИИ-агентов

В интерфейсе (раздел «Как пользоваться») встроен готовый **Системный промпт (Skill)** для LLM-агентов. 
Скопировав его, вы можете настроить любого AI-агента (например, в ChatGPT, Claude или локальном LLM) так, чтобы он:
1. Самостоятельно отправлял тестовые или реальные события на ваш `src`.
2. Формировал JSON-конфигурацию дашборда.
3. Кодировал её в Base64 и выдавал вам готовую прямую ссылку на визуализацию данных.
